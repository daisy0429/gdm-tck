

以下是gdm-tests项目中与**安装包下载、环境部署、环境可用性检查**相关的脚本文件，按职责分类：
gdm-tests项目： /Users/dpp/*project/newdb/cyphertck/gdm-tests


### 安装包下载

| 文件 | 职责 |
|---|---|
| `runtime/run-gdm-bin.sh` | 容器内运行时脚本：从 Hadess 下载 GDM 二进制包（curl/wget），校验 SHA256，解压/安装到 `/opt/gdm-server`，然后执行 |
| `gdm_tests/hadess.py` | Python 端：根据版本号解析 Hadess 仓库 URL（dev/rc/release 通道），用 HEAD/ranged GET 探测包是否可下载，构造 `gdm-{FILEVER}.tar.gz` 完整下载地址 |

### 环境部署（Docker Compose 编排）

| 文件 | 职责 |
|---|---|
| `compose/docker-compose.yml` | 分布式 3 节点部署（gdm1/gdm2/gdm3），挂载 `run-gdm-bin.sh` 和各节点 toml 配置 |
| `compose/docker-compose.standalone.yml` | 单节点部署（gdm-standalone） |
| `compose/configs/standalone.toml` | 单节点 GDM 运行配置 |
| `compose/configs/node1.toml` / `node2.toml` / `node3.toml` | 分布式各节点 GDM 运行配置 |
| `gdm_tests/server.py` | `ComposeServer` 类：封装 `docker compose up/down/restart/kill/stop/start` 等生命周期管理，向 compose 传递环境变量（镜像、下载地址、认证信息等） |
| `gdm_tests/config.py` | `Settings` 数据类 + `load_settings()`：从环境变量组装所有部署参数（模式、镜像、下载 URL、端口映射等），bin 模式下自动调用 `hadess.py` 解析下载地址 |

### 环境可用性检查

| 文件 | 职责 |
|---|---|
| `gdm_tests/health.py` | HTTP 健康探针：轮询 `/live`、`/livez`（存活）、`/ready`、`/readyz`（就绪）端点，判断 service_ready 状态；`wait_for_node_health()` 等待节点可用；`scrape_metrics()` 抓取 `/metrics` |
| `gdm_tests/bolt.py` | `BoltNodeClient.wait_connect()` 和 `BoltServerClient.wait_all_connectable()`：轮询 Bolt 端口执行 `RETURN 1` 验证 Cypher 连通性 |
| `gdm_tests/server.py:wait_ready()` | 对所有节点调用 `wait_for_node_health`，compose up 后等待整体就绪；`wait_for_recovery()` 用于 chaos 测试中的恢复等待 |
